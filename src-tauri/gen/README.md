# Tauri Mobile Configuration

This directory is auto-generated by `tauri android init` and `tauri ios init`.
Do not edit these files manually — they are regenerated on each init command.

## Setup

### Prerequisites

1. **Rust** — Install via [rustup](https://rustup.rs/)
2. **Tauri CLI** — `npm install` (includes `@tauri-apps/cli` as a dev dependency)
3. **Android Studio** — For Android builds (install NDK & Android SDK)
4. **Xcode** — For iOS builds (macOS only)

### Android Setup

```bash
# Initialize Android project (generates gen/android/)
npm run tauri:android:init

# Run on connected Android device/emulator
npm run tauri:android:dev

# Build release APK
npm run tauri:android:build
```

### iOS Setup

```bash
# Initialize iOS project (generates gen/apple/)
npm run tauri:ios:init

# Run on connected iOS device/simulator
npm run tauri:ios:dev

# Build release IPA
npm run tauri:ios:build
```

## Architecture

```
src-tauri/
├── Cargo.toml          # Rust dependencies (tauri, notification plugin)
├── tauri.conf.json     # Tauri app configuration
├── capabilities/
│   └── default.json    # Permission grants (notifications, etc.)
├── src/
│   ├── main.rs         # Desktop entry point
│   └── lib.rs          # Shared lib with notification plugin
└── gen/                # Auto-generated mobile scaffolding
    ├── android/        # Android Studio project
    └── apple/          # Xcode project
```

## Notification Flow

```
Reminder fires → reminder-store.tick()
  → showBrowserNotification() (notification-utils.ts)
    → tauri-bridge.showNotification()
      → [Tauri] @tauri-apps/plugin-notification → native OS notification
      → [Web]   Notification API → browser notification
```

The `tauri-bridge.ts` module auto-detects the runtime and routes to the
appropriate notification backend. No code changes are needed when switching
between web, desktop, and mobile builds.
